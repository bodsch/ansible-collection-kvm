# Ansible Role - bodsch.kvm.storage_pool

Manage libvirt **storage pools** on a KVM host.

The pools are reconciled with the native
[`bodsch.kvm.libvirt_pool`](../../plugins/modules/libvirt_pool.py) module (no
`community.libvirt`): a single task defines every pool from structured
parameters, builds the target directory, and reconciles its run state and
autostart flag.

## Requirements

* libvirt installed and its daemons running (use the `bodsch.kvm.libvirt` role first).
* `libvirt-python` on the managed host.

## Role variables

| Variable | Default | Description |
| :-- | :-- | :-- |
| `storage_pools` | `[]` | list of pools to manage (see below) |

Each item of `storage_pools`:

| Key | Required | Description |
| :-- | :-- | :-- |
| `name` | yes | pool name |
| `path` | yes¹ | absolute target directory (`type=dir` pool) |
| `state` | no (`present`) | `present` (define), `active` (define + start), `inactive` (define + stop), `absent` (stop + undefine) |
| `autostart` | no | start the pool on host boot |
| `mode` | no | octal permissions of the target dir, e.g. `"0711"` |
| `owner` / `group` | no | numeric uid / gid of the target dir |
| `uuid` | no | fixed pool UUID (generated by libvirt when omitted) |

¹ `path` is required to *define* a new pool; it is ignored for an existing pool
(the definition is never rewritten) and not needed for `state: absent`.

## Example

```yaml
- hosts: kvm_hosts
  become: true
  roles:
    - role: bodsch.kvm.storage_pool
  vars:
    storage_pools:
      - name: pool
        path: /var/lib/libvirt/pool
        state: active
        autostart: true
        mode: "0711"
      - name: scratch
        state: absent
```
